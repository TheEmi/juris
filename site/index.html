🧠 Conway's Rules

Underpopulation:
Live cell with <2 neighbors dies
Survival:
Live cell with 2-3 neighbors survives
Overpopulation:
Live cell with >3 neighbors dies
Reproduction:
Dead cell with 3 neighbors becomes alive
🚫 Why Other Frameworks Can't Do This
❌
React/Vue/Angular: Cannot progressively enhance existing HTML at this scale. They require complete component rewrites and virtual DOM overhead for 600 interactive elements.
❌
jQuery/Vanilla JS: Would require massive manual DOM manipulation, event binding, and state synchronization code. Performance would degrade rapidly with complex state changes.
❌
Svelte/SolidJS: Compile-time frameworks that require build processes and can't enhance arbitrary existing HTML without complete reconstruction.
❌
Web Components: Too heavyweight for this scale, poor progressive enhancement story, and complex state management across 600 custom elements.
🎯 Why We Chose HTML Divs Over Canvas
✅
Progressive Enhancement Proof: Canvas starts blank - no progressive enhancement possible. HTML divs exist and work without JavaScript, then get enhanced.
✅
Accessibility: Each cell is a real DOM element with proper focus, hover states, and screen reader support. Canvas is a black box to assistive technology.
✅
Individual Reactivity: Each of the 600 cells updates independently based on state changes. Canvas requires full redraws - JurisJS updates only changed cells.
✅
CSS Integration: Cells use real CSS for styling, animations, and responsive design. Canvas requires manual pixel-level graphics programming.
✅
Inspect-able Elements: Open DevTools and inspect any cell - see its state, click handlers, and CSS. Canvas content is invisible to developer tools.
🚀 The JurisJS Advantage
⚡
Surgical Updates: Only cells that change state get DOM updates. 590 static cells + 10 changing cells = only 10 DOM operations per generation.
⚡
Zero Overhead: No virtual DOM diffing, no component lifecycle, no build process. Just direct enhancement of existing HTML.
⚡
Intentional Reactivity: State changes trigger updates only where needed. Static cells remain static, reactive cells update automatically.
⚡
Enterprise Scale: This same pattern works for 10 elements or 10,000 elements. Perfect performance scaling with complexity.

2. JurisJS Enhancement

Add reactivity with the enhance() API

const app = new Juris();

// Enhance the button with click behavior
app.enhance('#increment-simple', (props, { getState, setState }) => ({
  onClick: () => {
    const current = getState('counter', 0);
    setState('counter', current + 1);
  }
}));

// Enhance the display with reactive content
app.enhance('#simple-count', (props, { getState }) => ({
  textContent: () => getState('counter', 0),
  style: () => ({
    color: getState('counter', 0) > 5 ? '#22c55e' : '#c4b5fd'
  })
}));
